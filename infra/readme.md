# 🧩 Terraform Files Explained

## providers.tf
- Defines Terraform version
- Configures AWS provider
- Sets AWS region

---

## variables.tf
- All configurable values:
  - MongoDB URI
  - VPC / subnet IDs
  - SES email
  - Bedrock model
- Keeps secrets out of code

---

## terraform.tfvars.example
- Template for required values
- Must be copied to `terraform.tfvars`

---

## networking.tf
- Creates VPC endpoints for:
  - SQS
  - SES
- Allows Lambdas to communicate privately inside AWS

---

## sqs.tf
- Creates queues:
  - `paper_queue`
  - `email_queue`
- Creates Dead Letter Queues (DLQ)
- Handles retry logic for failures

---

## s3.tf
- Creates S3 bucket
- Stores generated audio files (MP3)

---

## iam.tf
- Creates IAM role for Lambdas
- Grants permissions for:
  - SQS
  - SES
  - Bedrock
  - Polly
  - S3
  - CloudWatch logs

---

## lambdas.tf
- Defines all Lambda functions:
  - scraper
  - subscription
  - email
  - AI processor
- Configures:
  - memory
  - timeout
  - environment variables
  - VPC access
- Connects SQS triggers to Lambdas

---

## eventbridge.tf
- Creates scheduled triggers:
  - scraper (daily)
  - subscription emails (daily)
- Invokes Lambda automatically

---

## monitoring.tf
- Creates CloudWatch log groups
- Enables logging for each Lambda

---

## outputs.tf
- Outputs useful values:
  - Lambda names
  - Queue URLs
  - S3 bucket name

---

# ⚙️ Lambda Functions Explained

Each Lambda replaces part of the original Express backend.

---

## shared/db.js
- MongoDB connection helper
- Optimized for Lambda:
  - reuses connection
  - avoids reconnection overhead

---

## scraper-processor
**Replaces:** scraper controller + service

### What it does:
- Fetches papers from arXiv
- Stores them in MongoDB
- Sends new paper jobs to SQS

### Trigger:
- EventBridge (scheduled)

---

## subscription-processor
**Replaces:** email controller logic

### What it does:
- Finds users and their subscriptions
- Matches relevant papers
- Sends email jobs to SQS

### Trigger:
- EventBridge (scheduled)

---

## email-sender
**Replaces:** email service

### What it does:
- Sends emails using SES
- Formats email content

### Trigger:
- SQS (email queue)

---

## ai-processor
**New (no Express equivalent)**

### What it does:
1. Generates summary using Bedrock
2. Converts summary to audio using Polly
3. Uploads audio to S3
4. Updates MongoDB with:
   - summary
   - audio URL

### Trigger:
- SQS (paper queue)

---

# ☁️ AWS Services Used

## AWS Lambda
- Runs backend logic
- Replaces Express server

---

## Amazon SQS
- Message queues for async processing
- Decouples services
- Handles retries

---

## Amazon EventBridge
- Schedules jobs (cron)
- Triggers Lambdas automatically

---

## Amazon SES
- Sends emails to users

---

## Amazon S3
- Stores audio files generated by Polly

---

## Amazon Polly
- Converts text → speech (MP3)

---

## Amazon Bedrock
- Generates AI summaries of papers

---

## AWS IAM
- Controls permissions for Lambdas

---

## Amazon CloudWatch
- Logs and monitoring for Lambdas

---

# 🔄 System Flow (Simplified)

1. EventBridge → scraper Lambda
2. scraper → MongoDB + SQS
3. SQS → AI processor Lambda
4. AI processor → Bedrock + Polly + S3 + MongoDB
5. EventBridge → subscription Lambda
6. subscription → SQS
7. SQS → email Lambda
8. email Lambda → SES

---

# ⚠️ Notes

- Backend is now split into multiple services
- Communication happens through SQS instead of direct calls
- System is asynchronous and scalable
